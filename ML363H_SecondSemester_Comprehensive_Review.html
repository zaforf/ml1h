I. Logistic Regression Foundations (Pages 1–2)

Page 1: Probabilistic Binary Classification

The starting point is binary classification with labels \(y \in \{-1, +1\}\). Instead of predicting only a hard label, logistic regression models a conditional probability:

\[ \Pr(y=+1 \mid x) = \sigma(w^T x), \qquad \Pr(y=-1 \mid x) = \sigma(-w^T x), \] where \[ \sigma(z) = \frac{1}{1+e^{-z}}. \]

The identity \(\sigma(z)+\sigma(-z)=1\) guarantees these two probabilities are complementary. The notes also include \(f(x)=\operatorname{sign}(w\cdot x)\), which is the corresponding hard classifier induced by the same linear score.

Likelihood Objective

Given training data \(\{(x^1,y^1),\dots,(x^m,y^m)\}\), assume examples are conditionally independent given \(w\). Then likelihood is:

\[ L(w)=\prod_{i=1}^m \Pr(y^i\mid x^i,w)=\prod_{i=1}^m \sigma\!\big(y^i w^T x^i\big). \]

Taking logs converts products to sums:

\[ \log L(w) = \sum_{i=1}^m \log \sigma\!\big(y^i w^T x^i\big). \]
Main idea: learning logistic regression means choosing \(w\) that makes observed labels most probable under this model.

Page 2: MLE, Logistic Loss, and Convex Optimization

Page 2 turns the log-likelihood maximization into a minimization problem:

\[ \max_w \sum_{i=1}^m \log \sigma(y^i w^T x^i) \quad\Longleftrightarrow\quad \min_w -\sum_{i=1}^m \log \sigma(y^i w^T x^i). \]

Using \(\sigma(z)=\frac{1}{1+e^{-z}}\), one gets the standard logistic loss form:

\[ -\log \sigma(z)=\log(1+e^{-z}), \] so \[ \min_w \sum_{i=1}^m \log\!\big(1+e^{-y^i w^T x^i}\big). \]
Optimization consequence: this objective is convex in \(w\), so gradient-based optimization can find a global optimum (up to numerical tolerance), not just a local one.

Transition to Neural Networks

The notes then generalize from one logistic unit to multi-unit compositions. The expression written in the transcript is:

\[ \Pr(y\mid x)=\sigma\!\left(\sum_{i=1}^{m} a_i\,\sigma(w_i^T x)\right). \]

Conceptually, this is a one-hidden-layer network: hidden activations \(\sigma(w_i^T x)\), then a weighted sum with coefficients \(a_i\), then final sigmoid output.

III. Backpropagation and Chain Rule Mechanics (Pages 4–5)

Page 4: Gradient Descent and Backpropagation via Chain Rule

Page 4 makes the algorithmic step explicit: define a loss \(L(f_w(x),y)\), then minimize it with gradient descent.

\[ w \leftarrow w - \eta \nabla_w L(f_w(x^i),y^i). \]

The core technical tool is backpropagation, i.e., efficient gradient computation in layered compositions.

Multivariate Chain Rule (Central Formula)

If \(f\) depends on intermediate variables \(U_1,\dots,U_k\), and each \(U_j\) depends on \(z\), then: \[ \frac{\partial f}{\partial z} = \sum_{j=1}^{k} \frac{\partial f}{\partial U_j} \cdot \frac{\partial U_j}{\partial z}. \] This is the exact rule used repeatedly along a computational graph.

Computational Graph Interpretation

The sketch on this page indicates variables built in stages (e.g., sums, exponentials, hidden intermediates \(z_1,z_2\)), and asks for gradients such as \(\frac{\partial f}{\partial x}\), \(\frac{\partial f}{\partial y}\). Backprop computes these by propagating derivatives from output to inputs through local Jacobian factors.

Training pipeline from Pages 1–4: choose a probabilistic model \(\rightarrow\) define loss (negative log-likelihood or square loss) \(\rightarrow\) compute gradients via chain rule/backprop \(\rightarrow\) update parameters by gradient descent.

Page 12: Random-Walk View, Expectation Drift, and Azuma

Define indicator variables at depth \(i\):

\[ Y_i = \begin{cases} 1,& \text{if }x\text{ moves right at depth }i,\\ 0,& \text{otherwise.} \end{cases} \]

Then \(S_T=\sum_{i=1}^T Y_i\) counts right-moves along the path. By linearity:

\[ \mathbb{E}[S_T] = \sum_{i=1}^T \mathbb{E}[Y_i]. \]

Under the balanced condition for positive examples, \(\mathbb{E}[Y_i]\approx \frac12+\gamma\), so:

\[ \mathbb{E}[S_T] = T\left(\frac12+\gamma\right)=\frac{T}{2}+\gamma T. \]

The next proof target is to show \[ \Pr\!\left[S_T \ge \frac{T}{2}\right] \] is large (for positive points), i.e., path tends to the correct side with high probability. The page notes that a martingale setup plus Azuma's inequality gives concentration around this positive drift.

High-level implication: expected drift by \(\gamma T\) plus concentration yields a correctness probability that increases with depth, and this produces the \(T = \tilde{O}(1/\gamma^2 \log(1/\epsilon))\) dependence shown on prior pages.

X. Spectral PCA Optimization and Power Method (Pages 29–32)

Page 29: Spectral Theorem Form of PCA Objective

The page explicitly invokes the spectral theorem for symmetric matrices: \[ A=QDQ^T,\quad Q^TQ=I,\quad D=\operatorname{diag}(\lambda_1,\dots,\lambda_n). \] For covariance-type \(A\), eigenvalues satisfy \(\lambda_1\ge\lambda_2\ge\cdots\ge\lambda_n\ge 0\).

Page 32: Why Power Method Converges and Iteration Complexity

Using \(A=QDQ^T\), the notes write \(A^t=QD^tQ^T\). Expand initial unit vector in eigenbasis: \[ u_0=\sum_{i=1}^n c_iV_i. \] Then \[ A^tu_0=\sum_{i=1}^n c_i\lambda_i^tV_i =\lambda_1^t\!\left(c_1V_1+\sum_{i\ge 2}c_i\left(\frac{\lambda_i}{\lambda_1}\right)^tV_i\right). \] Assuming \(\lambda_1>\lambda_2\ge\lambda_3\ge\cdots\), the ratio terms decay to 0, so \[ A^tu_0\approx \lambda_1^tc_1V_1. \] After normalization: \[ \frac{A^tu_0}{\|A^tu_0\|}\to \frac{V_1}{\|V_1\|}, \] i.e., convergence to top eigenvector direction.
